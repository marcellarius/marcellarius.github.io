Shell Database

The goal of this project was to modernise the record keeping for a large sea shell collection. A specimen is worthless without details of when and where it was collected. Furthermore, scientific consensus on taxonomy changes as more research is done, so it often becomes necessary to update the classification of specimens in the database.

This was previously done with paper records, and while much of the data had been shifted into Word documents, this was prone to errors and difficult to search. Past attempts at setting up an Access database had been rejected because they were too hard to use.

To solve this, a modern web application was designed and developed that would make it easy to add and update records, while helping to ensure consistent spellings and avoiding accidental data loss.

The database stores a taxonomic tree and efficiently generates all scientific names from this. This normalisation made it much easier to build tools to tidy up the taxonomy, such as moving entire genera between families.

Online Wargame

This is a computer adaptation of a turn-based board game simulating World War 2 at operational scale. A large scenario will have hundreds or thousands of units in play, each unit is a little cardboard chit that get stacked into awkward piles on a vast map.

This style of game appeals to a particular hardcore military simulation buff but the impracticality of these large-scale games, and sparse network of players makes computer play-aids or games quite appealing.

We aimed to make a more modern solution that captured the aesthetic of the tabletop games, had flexible and easy multiplayer and a complete rules implementation.

Challenges included visualising and communicating information to the players, balancing between fidelity to the board game and what works in a turn-based computer game, and keeping game servers lightweight to reduce operating costs.

The game was developed in partnership with a Scottish game developer and the editing toolchain we created could produce output for large-format printing.

Map Editor

We created a bespoke editing tool which allowed us to use the same artwork for the computer game and print. Within 8 months we had built this tool from scratch, got it into the hands of our mappers, and met the optimistic deadlines for printing.

Noteworthy features

  • Imported coastline data from GIS databases
  • Choice of several map projections + rotation to find the best framing for the area you want to map
  • All the niceties you'd expect: multi-step undo/redo (command based), autosave
  • MVC architecture, allowing code reuse in game
  • Fast and fluid: hardware accelerated graphics + efficient updates kept it responsive
  • Sweet zoom and pan controls make moving around a breeze
  • Map out large area, then define map sections for cropping and export
  • Deformer tools to non-destructively fettle coastlines to fit a hex grid
  • File format backwards compatibility: we needed to deliver fast and iterate, compatibility was high priority
  • SVG Export: high quality vector output to render for print
