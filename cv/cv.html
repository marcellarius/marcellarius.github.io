Recent Projects

Sea Shell Database (2015-2017)

This project delivers a web application to manage records for a large private sea-shell collection. I built this application from scratch to move specimen records into a well-structured database. The application uses Flask and PostgreSQL. Most of the logic is server-side using Bootstrap 3 and Backbone.js in the front-end.

  • Scraping valuable specimen details from typo-filled text.
  • Providing a fast, easy-to-use interface for non-technical users
  • Trees in a relational DB, with efficient subtree queries
  • Acquire domain knowledge about zoological taxonomy
  • Interfacing with third-party web APIs
  • Full text search, fuzzy/partial matching, typeahead search UI
  • Generating printable specimen labels using HTML/CSS

Job Tracking System (2016)

I put this web app together for Eastbay Metering's internal use to track jobs, scanned paperwork, and digital photos. It is a responsive single-page application using Bootstrap, Backbone.js, and a RESTful JSON API provided by a Flask/PostgreSQL back-end.

  • Fast, mobile-friendly interface (SPA, history API)
  • Real-time updates, activity event logging
  • File and photo uploads, thumbnailing, drag and drop
  • Flexible admin UI for creation of new job types and actions

Online Computer Wargame (2010-2014)

I worked in a small team developing a computer adaptation of a WW2 wargame. We developed this in C# using Direct3D for graphics. My work on this project covered all areas of the game's development including design and technology choices, architecture, graphics, UI, networking, tooling, and web. The editing tools we created also produced maps for professional printing for the board game version.

  • Building editing tools for creating huge stylised maps
  • Exporting maps for professional printing
  • Understanding and implementing a complex rule set
  • Nested state-machine architecture for UI and game logic
  • Asynchronous network play using cloud-based servers
  • Working with XML. Serialization, custom code generators
  • Integrating the Chromium browser for the game UI
  • Embedded Lua VM and API for scenario scripting
  • .NET performance profiling and analysis

Academic

I studied computer science at the University of Waikato from 2006–2010 after receiving a scholarship during high school.

My academic interests were software development, graphics and video games, low-level programming, operating systems, and programming languages. I also took cognitive and perceptual psychology courses through to third year.

Starting in my second year I worked as a sessional assistant for the computer science department. This involved helping students understand and debug lab work, marking exercises, and supervising tests.

As part of a summer research placement in the WAND group, I was directly involved with setting up the University's new cluster computer. My work on this included researching, benchmarking, and deploying a cluster filesystem (GlusterFS); as well as learning, packaging, and setting up a job scheduling system (PBS).

My honours research project was making a distributed model checker that ran on the cluster, using the combined processing and memory of many computers to verify huge state spaces. My real interest was in the software development aspect rather formal methods research (to my supervisor's disappointment).

At the end of 2009 I took an opportunity to work full-time on a video game project, and I studied part-time in 2010. I am two first-year papers short of being eligible to graduate with a BCMS(Hons).

During university I took part in several programming competitions, both NZPC and ACM-ICPC South Pacific; as well as machine learning challenges.

I play the drums and bass guitar, enjoy coffee, cooking, tabletop RPGs, video games, cars and sim-racing. I'm comfortable repairing things and can brandish a screwdriver, spanner or soldering iron if needed.
